## **1. How to specify resources for container usage and update the configuration in the file?**

### ✅ YAML Configuration

```yaml
resources:
  requests:
    memory: "128Mi"
    cpu: "250m"
  limits:
    memory: "256Mi"
    cpu: "500m"
```

### ✅ Interview Answer (Paragraph)

To control container resource usage in Kubernetes, I define **resource requests and limits** in the deployment YAML file. Requests specify the minimum CPU and memory required for scheduling, while limits define the maximum resources a container can consume. This helps prevent resource contention and ensures efficient cluster utilization. I typically determine these values based on application profiling. This setup is also important for enabling autoscaling features like HPA, which relies on resource metrics to scale workloads dynamically.

## **2. Given container image (version 2.0), how do you update the configuration in the file?**

### ✅ YAML Update

```yaml
containers:
  - name: nginx
    image: nginx:2.0
```

### ✅ Command Alternative

```bash
kubectl set image deployment/nginx-deployment nginx=nginx:2.0
```

### ✅ Interview Answer (Paragraph)

To update a container image to a new version such as 2.0, I modify the `image` field in the Kubernetes deployment file and apply the changes using `kubectl apply`. Kubernetes performs a **rolling update**, gradually replacing old pods with new ones without downtime. I monitor the rollout using `kubectl rollout status` and can rollback if needed using `kubectl rollout undo`. In production, this process is automated via CI/CD pipelines to ensure controlled and reliable releases.

## **3. Write the kubelet commands for deploying image "karat\_studio\_image"?**

### ✅ Correct Commands (kubectl)

#### Create Deployment

```bash
kubectl create deployment karat-deployment --image=karat_studio_image
```

#### Update Image

```bash
kubectl set image deployment/karat-deployment karat-deployment=karat_studio_image
```

#### Verify Deployment

```bash
kubectl get deployments
kubectl rollout status deployment/karat-deployment
```

#### Rollback

```bash
kubectl rollout undo deployment/karat-deployment
```

***

### ✅ Interview Answer (Paragraph)

Kubelet itself does not provide commands to create or manage deployments, as it is a node-level agent responsible for running containers. Instead, we use `kubectl` to interact with the Kubernetes API server. To deploy an image like *karat\_studio\_image*, I would use `kubectl create deployment` or update an existing deployment using `kubectl set image`. Kubernetes then coordinates with kubelet to pull the image and run containers across nodes. This process supports rolling updates, rollback, and high availability.
