To use **AWS ECR** (Elastic Container Registry) and ensure you're always deploying the **latest image**, you need to update the `image` field in your Kubernetes Deployment YAML.

### ✅ Key changes:

1. Replace Docker Hub image (`nginx:1.26.1`) with your **ECR repository URI**
2. Use the `:latest` tag (or your preferred tag)
3. Optionally add `imagePullPolicy: Always` to ensure Kubernetes always pulls the latest image

***

## ✅ Updated `deployment.yaml`

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: nginx-deployment
  labels:
    app: nginx
spec:
  replicas: 3
  selector:
    matchLabels:
      app: nginx
  template:
    metadata:
      labels:
        app: nginx
    spec:
      containers:
      - name: nginx
        image: <AWS_ACCOUNT_ID>.dkr.ecr.<REGION>.amazonaws.com/nginx:latest
        imagePullPolicy: Always
        ports:
        - containerPort: 80
```

***

## 🔁 Replace placeholders:

* `<AWS_ACCOUNT_ID>` → Your AWS account ID (e.g., `123456789012`)
* `<REGION>` → AWS region (e.g., `ap-south-1` for Pune/Mumbai region)
* `nginx` → Your ECR repository name (if different)

**Example:**

```yaml
image: 123456789012.dkr.ecr.ap-south-1.amazonaws.com/nginx:latest
```

***

## ⚠️ Important Notes:

* Ensure your Kubernetes cluster has permission to pull from ECR:
  * On **EKS** → IAM roles handle this automatically
  * On other clusters → you must create an `imagePullSecret`
* `latest` tag is convenient but **not recommended for production** (better to use versioned tags like `v1.0.3` for traceability)

***

## ✅ Optional (for non-EKS clusters):

Add `imagePullSecrets`:

```yaml
spec:
  imagePullSecrets:
  - name: ecr-secret
```
