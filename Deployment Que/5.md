5)	What is security context and when to use it and where to use it?

### ✅ Interview‑Style Answer

**Security Context in Kubernetes** defines **privilege and access control settings** for a Pod or Container. It allows you to specify **how a container should run securely**, controlling permissions like user ID, file system access, and capabilities.

***

### ✅ Where to Use It

SecurityContext can be applied at **two levels**:

1. **Pod level (`spec.securityContext`)**
   * Applies to all containers in the pod

2. **Container level (`containers[].securityContext`)**
   * Overrides pod-level settings for a specific container

***

### ✅ When to Use It

You use SecurityContext when you want to:

* ✅ Run containers as **non-root user**
* ✅ Enforce **least privilege principle**
* ✅ Restrict container capabilities (like preventing privilege escalation)
* ✅ Control file system permissions (read-only root FS)
* ✅ Meet **security compliance (e.g., CIS benchmarks, PCI, etc.)**

***

### ✅ Common SecurityContext Settings

#### 1. Run as non-root user

```yaml
securityContext:
  runAsUser: 1000
  runAsNonRoot: true
```

***

#### 2. Prevent privilege escalation

```yaml
securityContext:
  allowPrivilegeEscalation: false
```

***

#### 3. Read-only filesystem

```yaml
securityContext:
  readOnlyRootFilesystem: true
```

***

#### 4. Drop Linux capabilities

```yaml
securityContext:
  capabilities:
    drop:
      - ALL
```

***

#### 5. Set file system group (for volumes)

```yaml
securityContext:
  fsGroup: 2000
```

***

### ✅ Example (Integrated in Your Deployment)

```yaml
spec:
  securityContext:   # Pod level
    runAsNonRoot: true
    fsGroup: 2000

  containers:
  - name: nginx
    image: nginx:1.26.1
    securityContext:   # Container level
      runAsUser: 1000
      allowPrivilegeEscalation: false
      readOnlyRootFilesystem: true
```

***

### ✅ Why It Is Important

* 🔒 Improves **container security**
* 🚫 Prevents running as root (common attack vector)
* 🛡️ Limits damage if container is compromised
* ✅ Helps meet **enterprise security standards**

***

### ✅ Strong Closing Line

> SecurityContext is used to enforce security best practices in Kubernetes by controlling how containers run, ensuring least privilege, reducing attack surface, and improving overall workload security.
