	Given container sha:xxxxxxxxxnumberxxx and how to update the configuration and sure we use the SHA value 

To ensure you're using a **specific container image immutably**, Kubernetes best practice is to use the **image SHA digest** instead of a tag.

***

# ✅ Why use Image SHA (Digest)?

* Tags like `nginx:1.26.1` can be changed behind the scenes
* SHA digest guarantees **exact image version**
* Prevents unexpected changes in deployments

***

# ✅ Given SHA

Example:

```
sha256:xxxxxxxxxnumberxxx
```

***

# ✅ How to Update the Deployment YAML

Replace:

```yaml
image: nginx:1.26.1
```

With:

```yaml
image: nginx@sha256:xxxxxxxxxnumberxxx
```

***

# ✅ Updated Deployment Snippet

```yaml
containers:
- name: nginx
  image: nginx@sha256:xxxxxxxxxnumberxxx
  imagePullPolicy: IfNotPresent
  ports:
  - containerPort: 80
```

***

# ✅ Important Notes

### ✅ 1. Correct Format

Always use:

```bash
<image-name>@sha256:<digest>
```

❌ Wrong:

```bash
nginx:sha256:xxxx
```

✅ Correct:

```bash
nginx@sha256:xxxx
```

***

### ✅ 2. imagePullPolicy Recommendation

When using SHA:

```yaml
imagePullPolicy: IfNotPresent
```

👉 Because SHA is immutable, no need to always pull

***

### ✅ 3. Deployment Update Command

If updating an existing deployment:

```bash
kubectl set image deployment/nginx-deployment nginx=nginx@sha256:xxxxxxxxxnumberxxx
```

***

### ✅ 4. Verify Deployment

```bash
kubectl rollout status deployment/nginx-deployment
kubectl get pods -o wide
```

***

# ✅ Bonus: How to Get Image SHA

```bash
docker pull nginx:1.26.1
docker inspect --format='{{index .RepoDigests 0}}' nginx:1.26.1
```

Example output:

```
nginx@sha256:abc123...
```
