3)	How will you pass the environment variables/ configurational data to your deployment

In Kubernetes, you typically pass **environment variables or configuration data** to a Deployment in a few standard ways:

***

# ✅ 1. Using `env` (Direct Environment Variables)

You can define environment variables directly inside the container spec:

```yaml
containers:
- name: nginx
  image: nginx:1.26.1
  env:
    - name: ENVIRONMENT
      value: "production"
    - name: APP_NAME
      value: "nginx-app"
```

👉 Good for **simple/static values**.

***

# ✅ 2. Using ConfigMap (Best for non-sensitive config)

### Step 1: Create ConfigMap

```yaml
apiVersion: v1
kind: ConfigMap
metadata:
  name: nginx-config
data:
  ENVIRONMENT: "production"
  APP_MODE: "debug"
```

### Step 2: Reference in Deployment

#### Option A: Inject as env variables

```yaml
envFrom:
  - configMapRef:
      name: nginx-config
```

#### Option B: Individual keys

```yaml
env:
  - name: ENVIRONMENT
    valueFrom:
      configMapKeyRef:
        name: nginx-config
        key: ENVIRONMENT
```

👉 Best when you want **centralized, reusable configuration**.

***

# ✅ 3. Using Secrets (For sensitive data)

### Step 1: Create Secret

```yaml
apiVersion: v1
kind: Secret
metadata:
  name: nginx-secret
type: Opaque
data:
  PASSWORD: cGFzc3dvcmQ=   # base64 encoded
```

### Step 2: Use in Deployment

```yaml
env:
  - name: PASSWORD
    valueFrom:
      secretKeyRef:
        name: nginx-secret
        key: PASSWORD
```

👉 Use for:

* passwords
* API keys
* tokens

***

# ✅ 4. Mount ConfigMap/Secret as Files

Instead of env vars, mount them as files:

```yaml
volumes:
  - name: config-volume
    configMap:
      name: nginx-config

containers:
- name: nginx
  volumeMounts:
    - name: config-volume
      mountPath: /etc/config
```

👉 Files will appear inside container:

```
/etc/config/ENVIRONMENT
/etc/config/APP_MODE
```

***

# ✅ 5. Using Downward API (Pod/Cluster metadata)

Useful for passing Kubernetes metadata:

```yaml
env:
  - name: POD_NAME
    valueFrom:
      fieldRef:
        fieldPath: metadata.name
```

***

# ✅ Example (Integrated into Your Deployment)

Here’s how your deployment would look with ConfigMap + Secret:

```yaml
containers:
- name: nginx
  image: nginx:1.26.1

  envFrom:
    - configMapRef:
        name: nginx-config

  env:
    - name: DB_PASSWORD
      valueFrom:
        secretKeyRef:
          name: nginx-secret
          key: PASSWORD
```

***

# ✅ Best Practices

* ✅ Use **ConfigMap** for non-sensitive config
* ✅ Use **Secrets** for sensitive data
* ✅ Avoid hardcoding env values in YAML
* ✅ Prefer `envFrom` for bulk injection
* ✅ Use volume mounts when app expects config files
