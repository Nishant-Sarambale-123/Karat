•	What is the image registry its uses 

An **Image Registry** is a central place where you **store, manage, and distribute container images** (like Docker images). It is a key component in DevOps and Kubernetes workflows.

***

# ✅ What is an Image Registry?

👉 An **Image Registry** is a repository that stores **container images** so they can be pulled and used by Kubernetes, Docker, or other platforms.

📌 Example:

* You build a Docker image → push it to registry → Kubernetes pulls it to run containers

***

# ✅ Architecture Flow

```
Developer → Build Image → Push to Registry → Kubernetes pulls → Runs Pod
```

***

# ✅ Types of Image Registries

### 🔹 Public Registries

* Docker Hub
* GitHub Container Registry

### 🔹 Private Registries

* Azure Container Registry (ACR)
* Amazon ECR
* Google GCR
* Self-hosted (Harbor, Nexus)

***

# ✅ How It Works (Step-by-Step)

### 1. Build Image

```bash
docker build -t my-app:1.0 .
```

### 2. Tag Image

```bash
docker tag my-app:1.0 myregistry/my-app:1.0
```

### 3. Push to Registry

```bash
docker push myregistry/my-app:1.0
```

### 4. Pull from Registry (Kubernetes)

```yaml
containers:
  - name: my-app
    image: myregistry/my-app:1.0
```

***

# ✅ Uses of Image Registry

### 🔹 1. Central Storage

* Stores all application images in one place

### 🔹 2. Version Control

* Keeps multiple versions (tags like v1, v2, latest)

### 🔹 3. CI/CD Integration

* Pipelines push images automatically after build

### 🔹 4. Deployment

* Kubernetes pulls images from registry to create pods

### 🔹 5. Security

* Scan images for vulnerabilities
* Control access (private repositories)

### 🔹 6. Reusability

* Same image used across dev, QA, and prod

***

# ✅ Real DevOps Example

1. Developer commits code
2. CI pipeline builds Docker image
3. Image pushed to **Azure Container Registry**
4. Kubernetes pulls image and deploys in **namespace**
5. Application runs as a pod

***

***

# ✅ Common Issues & Fixes

| Issue               | Solution                        |
| ------------------- | ------------------------------- |
| Image pull error    | Check image name & tag          |
| Unauthorized access | Configure credentials / secrets |
| Old image used      | Use correct version/tag         |
| Slow pulls          | Use local/private registry      |
