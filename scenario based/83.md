Here’s a simple and interview-friendly **Bash script using environment variables**:

```bash
#!/bin/bash

# Read environment variables
APP_NAME=${APP_NAME:-"MyApp"}     # default value if not set
ENVIRONMENT=${ENVIRONMENT:-"dev"}
LOG_LEVEL=${LOG_LEVEL:-"INFO"}

echo "Starting application..."
echo "App Name: $APP_NAME"
echo "Environment: $ENVIRONMENT"
echo "Log Level: $LOG_LEVEL"

# Example logic using env variables
if [ "$ENVIRONMENT" = "prod" ]; then
    echo "Running in production mode"
else
    echo "Running in non-production mode"
fi

# Simulate app start
echo "Application $APP_NAME started successfully!"
```

***

## ✅ How to run it

### 1. Set environment variables

```bash
export APP_NAME=OrderService
export ENVIRONMENT=prod
export LOG_LEVEL=DEBUG
```

### 2. Run script

```bash
bash script.sh
```

***

## 💡 Key Points (good for interview)

*   Uses environment variables for **configuration instead of hardcoding**
*   Supports **default values** (`${VAR:-default}`)
*   Helps make scripts **flexible, reusable, and environment-independent**
*   Commonly used in **CI/CD pipelines, Docker, and deployments**
