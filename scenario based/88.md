What is context manager in Python (usage in automation)?

A **context manager** in Python is a feature that helps manage resources (like files, network connections, locks) efficiently by automatically handling setup and cleanup using the `with` statement. It ensures that resources are properly released, even if an error occurs.


***

## ✅ Simple Example

```python
with open("file.txt", "r") as f:
    data = f.read()
```

👉 File is automatically closed after the block — no need to call `f.close()`

***

## ✅ Usage in Automation (Important for interviews)

In automation/server tasks, context managers are used to safely handle resources:

### 1. File Handling (logs, backups)

```python
with open("log.txt", "a") as f:
    f.write("New log entry\n")
```

***

### 2. Database Connections

```python
with db.connect() as conn:
    conn.execute("SELECT * FROM users")
```

👉 Automatically closes connection

***

### 3. Lock Management (thread-safe scripts)

```python
from threading import Lock

lock = Lock()

with lock:
    print("Thread-safe operation")
```

***

### 4. Network / API Calls

```python
import requests

with requests.Session() as session:
    response = session.get("https://api.example.com")
```
