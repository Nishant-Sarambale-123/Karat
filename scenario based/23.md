How to execute commands inside a running container?

Here’s a **clear interview-style answer**:

***

**“To execute commands inside a running container in Kubernetes, we use the `kubectl exec` command. This allows us to run commands or open an interactive shell within a container for debugging or inspection.”**

### ✅ Basic Command:

```bash
kubectl exec <pod-name> -- <command>
```

### ✅ Example:

```bash
kubectl exec my-pod -- ls /
```

### ✅ Interactive Shell:

```bash
kubectl exec -it <pod-name> -- /bin/sh
```

or

```bash
kubectl exec -it <pod-name> -- /bin/bash
```

### ✅ For Multi-Container Pods:

```bash
kubectl exec -it <pod-name> -c <container-name> -- /bin/sh
```

***

### 🎯 Interview Closing Line:

**“So, we use `kubectl exec`, typically with `-it`, to run commands or access the terminal inside a running container for troubleshooting and operational tasks.”**
