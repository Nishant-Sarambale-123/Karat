How to use Terraform modules for scalability?


To use **Terraform modules for scalability**, the idea is to write your infrastructure code **once and reuse it multiple times** across different environments or services.

In Terraform, a **module** is a reusable piece of code that defines a specific resource or set of resources (like VPC, EC2, database). Instead of duplicating code, you create a module and then call it wherever needed.

### How it helps with scalability:

*   You can **reuse the same module** for dev, staging, and prod by just changing input variables.
*   It keeps your code **clean, organized, and easy to maintain**.
*   If you need to update something (like a security rule), you **change it in one place**, and it applies everywhere.

### Example:

**Step 1: Create a module (modules/ec2/main.tf)**

```hcl
resource "aws_instance" "example" {
  ami           = var.ami
  instance_type = var.instance_type
}
```

**Step 2: Define variables (modules/ec2/variables.tf)**

```hcl
variable "ami" {}
variable "instance_type" {}
```

**Step 3: Use module in environment (dev/main.tf)**

```hcl
module "ec2_dev" {
  source         = "../modules/ec2"
  ami            = "ami-123456"
  instance_type  = "t2.micro"
}
```

**Step 4: Use same module in prod (prod/main.tf)**

```hcl
module "ec2_prod" {
  source         = "../modules/ec2"
  ami            = "ami-789012"
  instance_type  = "t2.large"
}
```

### Best Practices:

*   Keep modules **small and focused** (one responsibility)
*   Use **input variables and outputs**
*   Version your modules (especially if stored remotely)
*   Store modules in a **shared repo or registry**

### Interview Summary:

Terraform modules improve scalability by enabling **code reuse, consistency, and easier maintenance**. Instead of duplicating infrastructure code, we create reusable modules and customize them using variables for different environments, making the system clean and scalable.
