74.md What is Helm folder structure?
The **Helm folder structure** defines how a Helm chart is organized so that it is easy to manage, reuse, and deploy applications in Kubernetes.

### 🔹 Standard Helm Chart Structure:

    my-chart/
      Chart.yaml
      values.yaml
      charts/
      templates/
      .helmignore

### 🔹 Explanation:

*   **Chart.yaml**  
    Contains metadata about the chart like name, version, and description.

*   **values.yaml**  
    Holds default configuration values that can be overridden during deployment.

*   **templates/**  
    Contains Kubernetes resource files (like Deployment, Service) written as templates with variables.

*   **charts/**  
    Stores dependent charts (subcharts) if your application has multiple components.

*   **.helmignore**  
    Works like `.gitignore` to exclude unnecessary files from the chart.

***

### ✅ Interview Summary:

The Helm folder structure consists of key files like **Chart.yaml for metadata, values.yaml for configuration, templates for Kubernetes manifests, and charts for dependencies**. This structure helps in organizing, templating, and managing Kubernetes applications efficiently.
