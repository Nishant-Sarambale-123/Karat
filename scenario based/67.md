How to store secrets in Terraform?

To store **secrets in Terraform**, the main goal is to **avoid hardcoding sensitive data** (like passwords, API keys) in your code and keep them secure.

### Recommended Approaches:

✅ **1. Use Secret Managers (Best Practice)**  
Store secrets in external tools and fetch them in Terraform:

*   AWS Secrets Manager
*   AWS SSM Parameter Store
*   Azure Key Vault
*   HashiCorp Vault

Example (AWS SSM):

```hcl
data "aws_ssm_parameter" "db_password" {
  name = "/myapp/db_password"
}

resource "aws_db_instance" "example" {
  password = data.aws_ssm_parameter.db_password.value
}
```

***

✅ **2. Use Environment Variables**  
You can pass secrets via environment variables instead of writing them in code:

```bash
export TF_VAR_db_password="mysecret"
```

Terraform:

```hcl
variable "db_password" {}
```

***

✅ **3. Use Encrypted Remote State**  
Store Terraform state in a secure backend (like S3 with encryption enabled).  
**Important**: State files can contain secrets, so:

*   Enable encryption (`encrypt = true`)
*   Restrict access using IAM

***

✅ **4. Use `.tfvars` Carefully (Not Recommended for Secrets)**  
You can store variables in `terraform.tfvars`, but avoid this for secrets unless:

*   File is encrypted
*   Not committed to Git (`.gitignore`)

***

✅ **5. Sensitive Variables Flag**  
Mark variables as sensitive to prevent them from being shown in logs:

```hcl
variable "db_password" {
  sensitive = true
}
```

***

### What NOT to do:

❌ Hardcoding secrets in `.tf` files  
❌ Committing secrets to Git  
❌ Storing plain text secrets in state files without protection

***

### Interview Summary:

In Terraform, secrets should be stored securely using **external secret managers (like AWS Secrets Manager or Vault)** and accessed dynamically. Avoid hardcoding secrets; instead, use **environment variables, encrypted remote state, and sensitive variables** to ensure security and compliance.
