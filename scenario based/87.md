Use Python scripts for server administration

You can schedule cron-like jobs in Python using libraries or by integrating with system cron.

***

# ✅ 1. Using `schedule` Library (Simple & Popular)

### Install:

```bash
pip install schedule
```

### Example:

```python
import schedule
import time

def job():
    print("Running scheduled job...")

# Run every 5 seconds
schedule.every(5).seconds.do(job)

# Run every day at specific time
# schedule.every().day.at("10:00").do(job)

while True:
    schedule.run_pending()
    time.sleep(1)
```

👉 Best for simple scripts

***

# ✅ 2. Using `cron` (Best in production)

You write a Python script and use system cron to schedule it.

### Step 1: Python script

```python
print("Cron job executed")
```

### Step 2: Add cron entry

```bash
crontab -e
```

Example:

```bash
* * * * * /usr/bin/python3 /path/script.py
```

👉 Runs every minute

✔ Most reliable approach  
✔ Used in real systems

***

# ✅ 3. Using `APScheduler` (Advanced)

### Install:

```bash
pip install apscheduler
```

### Example:

```python
from apscheduler.schedulers.blocking import BlockingScheduler

def job():
    print("Job running...")

scheduler = BlockingScheduler()
scheduler.add_job(job, 'interval', seconds=10)

scheduler.start()
```

👉 Good for:

*   Background jobs
*   Production apps

***

# 🎯 Interview Answer (Short Paragraph)

"We can schedule cron jobs in Python either by using libraries like `schedule` or `APScheduler` for in-code scheduling, or by using the system cron where a Python script is triggered using crontab. For production systems, using system cron is more reliable, while libraries are useful for lightweight or application-level scheduling."
