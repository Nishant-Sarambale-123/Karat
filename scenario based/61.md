**Terraform vs CloudFormation — when to choose which (simple interview answer):**

Terraform and CloudFormation are both Infrastructure as Code (IaC) tools, but the choice depends on your use case. **Terraform** is best when you are working with **multi-cloud or hybrid environments** because it supports AWS, Azure, GCP, and many other providers in a single tool, giving flexibility and avoiding vendor lock-in. It also has a large community and reusable modules.

On the other hand, **CloudFormation** is ideal when you are fully invested in **AWS**. It is a native AWS service, so it integrates deeply with AWS resources, provides better support for new AWS features, and has built-in security and consistency without needing external tools.

**In short:**

*   Choose **Terraform** for **multi-cloud, flexibility, and portability**
*   Choose **CloudFormation** for **AWS-only projects with tight integration and simplicity**
