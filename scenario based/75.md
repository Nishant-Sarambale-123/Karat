Commands to create and deploy Helm charts

In Helm, there are a few key commands used to **create, package, and deploy charts**.

To **create a new Helm chart**, we use:

```bash
helm create my-chart
```

This command generates a standard folder structure with templates and configuration files.

Before deploying, we usually **validate the chart** using:

```bash
helm lint my-chart
```

To **install (deploy) the chart** into Kubernetes:

```bash
helm install my-release ./my-chart
```

Here, `my-release` is the name of the deployment.

If we need to **customize values**, we use:

```bash
helm install my-release ./my-chart -f values.yaml
```

To **upgrade an existing release**:

```bash
helm upgrade my-release ./my-chart
```

If something goes wrong, we can **rollback**:

```bash
helm rollback my-release <revision-number>
```

To **view deployed releases**:

```bash
helm list
```

To **uninstall the release**:

```bash
helm uninstall my-release
```

### ✅ Interview Summary:

Helm provides simple commands like **helm create, helm install, helm upgrade, and helm rollback** to manage the full lifecycle of Kubernetes applications, making deployment and management easy and efficient.
