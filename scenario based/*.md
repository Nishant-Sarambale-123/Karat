6.	What are the techniques to scale the applications?

The main techniques to scale applications (especially in DevOps/Kubernetes) are:

**1. Vertical Scaling (Scale Up)**  
Increase the resources of a single machine, like adding more **CPU, RAM, or storage**. It’s simple but has a hardware limit.

**2. Horizontal Scaling (Scale Out)**  
Add more instances of the application (servers/pods). This is the most commonly used approach in cloud and Kubernetes because it provides **better availability and scalability**.

**3. Auto Scaling**  
Automatically adjust resources based on load. In Kubernetes, this is done using:

*   **HPA (Horizontal Pod Autoscaler)** – scales pods based on CPU/memory
*   **VPA (Vertical Pod Autoscaler)** – adjusts pod resources
*   **Cluster Autoscaler** – adds/removes nodes

**4. Load Balancing**  
Distribute incoming traffic across multiple servers/pods to avoid overload and improve performance.

**5. Caching**  
Use tools like Redis or CDN to reduce load on backend systems and improve response time.

**In short:**

*   Vertical → increase power
*   Horizontal → add more instances ✅ (most preferred)
*   Auto Scaling → dynamic scaling based on traffic 🚀
