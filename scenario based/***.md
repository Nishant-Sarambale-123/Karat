3.	how to implement immutable resources?

Immutable resources are implemented by **never modifying existing infrastructure after deployment**. Instead, whenever a change is needed, a **new version of the resource is created and the old one is replaced**. This ensures consistency and avoids configuration drift.

In practice, this is done using **Infrastructure as Code (IaC)** tools like Terraform or CloudFormation, where infrastructure is defined in code and recreated on updates. Teams also use **pre-built machine images** (like AMIs) or **container images (Docker)**, so every deployment is a fresh, identical instance.

Modern systems like **Kubernetes follow immutability by default**, where new containers are deployed and old ones are terminated. Deployment strategies like **blue-green or rolling updates** help switch traffic safely.

Overall, immutable infrastructure improves **reliability, consistency, and easy rollback**, making deployments more predictable.
