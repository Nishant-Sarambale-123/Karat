How to store secrets in bash scripts?


The key rule is: **never hardcode secrets directly in a Bash script** (like passwords, API keys).

Here are the **simple and correct ways**:

***

## ✅ 1. Use Environment Variables (Most common)

Store secrets outside the script and read them inside.

```bash
#!/bin/bash

echo "Using API key: $API_KEY"
```

### Set secret:

```bash
export API_KEY=my-secret-key
bash script.sh
```

✔ Safe and flexible  
✔ Used in CI/CD, Docker, Kubernetes

***

## ✅ 2. Use `.env` File (with restricted access)

**.env file (keep secure):**

    API_KEY=my-secret-key
    DB_PASSWORD=pass123

**Script:**

```bash
#!/bin/bash

source .env
echo "API Key: $API_KEY"
```

👉 Set permission:

```bash
chmod 600 .env
```

***

## ✅ 3. Use Secret Managers (Best for production)

Store secrets in tools like:

*   AWS Secrets Manager
*   Azure Key Vault
*   HashiCorp Vault

Then fetch inside script:

```bash
SECRET=$(vault read my-secret)
```

✔ Most secure  
✔ Used in real production systems

***

## ✅ 4. Use File with Restricted Permissions

Store secret in a file:

```bash
#!/bin/bash
PASSWORD=$(cat /secure/path/password.txt)
```

👉 Secure it:

```bash
chmod 600 password.txt
```

***

## ❌ What NOT to do

```bash
API_KEY="my-secret-key"   # ❌ hardcoded (bad practice)
```

***

## 🎯 Interview Answer (Short Paragraph)

"Secrets should not be hardcoded in Bash scripts. Instead, they should be stored securely using environment variables, .env files with restricted permissions, or secret management tools like Vault or cloud key stores. This approach improves security, makes scripts reusable across environments, and prevents sensitive data exposure."
