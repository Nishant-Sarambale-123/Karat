What are hard vs soft stage gates in CI/CD?



In CI/CD, **stage gates** are checkpoints that control whether a pipeline can move to the next step (like build → test → deploy).

👉 **Hard stage gates** are strict and mandatory. If a condition fails (like tests failing, security scan issues, or code quality not meeting standards), the pipeline **stops immediately** and does not proceed further. These are used where quality and safety are critical.

👉 **Soft stage gates** are more flexible. Even if a condition fails, the pipeline can still continue, but the issue is **flagged as a warning** or may require manual approval. These are used when you want visibility without blocking development.

***

## 🎯 Interview Answer (Simple Paragraph)

"In CI/CD, stage gates are checkpoints that control pipeline progression. Hard stage gates strictly enforce rules, meaning the pipeline stops if any condition like failed tests or security issues is not met. Soft stage gates, on the other hand, allow the pipeline to continue even if checks fail, usually with warnings or manual approvals. Hard gates ensure strict quality control, while soft gates provide flexibility during development."
